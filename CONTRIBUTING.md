# Contributing to Atlas

Atlas is currently an early-stage personal treasury product.

## Product rule

A feature is valuable only if it either:

1. reduces the number of manual financial decisions, or
2. improves the quality of those decisions.

## Data rule

The code is open source. User financial data is private and belongs to the user.

Atlas must stay local-first unless a future product decision explicitly changes that.

## Workflow

- Work from GitHub Issues.
- Keep changes small and reviewable.
- Prefer working software over abstract architecture.
- Every increment should be testable by the Product Owner.

## Definition of Done

A change is done only when the Product Owner can try it without reading code.
